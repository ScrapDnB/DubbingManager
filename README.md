**Dubbing Manager** — настольное приложение для подготовки проектов дубляжа: импорт сценариев, рабочие тексты, распределение актёров, телесуфлёр, видео, статистика и экспорт монтажных листов.

## Возможности

- **Проекты и файлы**: создание, сохранение, автосохранение, история недавних проектов, рабочая папка проекта и проверка потерянных файлов.
- **Рабочие тексты**: ASS, SRT и DOCX используются как источники импорта, а редактируемым источником после импорта становится JSON в `texts_dm`.
- **Актёры и роли**: проектная база актёров, глобальная база актёров, пол актёра, цвет актёра, глобальные и серийные назначения.
- **Перенос кастинга**: импорт и экспорт проектного распределения актёров для переноса между сезонами или проектами.
- **Главная таблица**: быстрый фильтр по актёру, фильтр пустых назначений, поиск, цвет актёра в ячейке и серийные переопределения ролей.
- **DOCX импорт**: гибкое сопоставление колонок, предпросмотр, сохранение настроек импорта и пересоздание рабочего текста из DOCX без повторного объединения реплик.
- **Телесуфлёр**: автопрокрутка, ручной скролл, OSC-синхронизация с Reaper, отдельное окно управления и переключение серий.
- **Экспорт**: HTML, Excel и Reaper RPP, предпросмотр HTML, предпросмотр RPP, настройка колонок и формата тайминга.
- **Бэкапы**: автоматические бэкапы рабочих текстов и глобальных настроек перед перезаписью.
- **Undo/Redo**: отмена и возврат основных действий.

## Быстрый старт

```bash
git clone https://github.com/ScrapDnB/DubbingManager.git
cd DubbingManager
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python main.py
```

Для headless-тестов:

```bash
QT_QPA_PLATFORM=offscreen .venv/bin/python -m pytest -q
```

## Основы работы

1. Создайте проект или откройте один из недавних.
2. Нажмите **Папка** и выберите рабочую папку проекта, если файлы лежат рядом.
3. Импортируйте ASS, SRT или DOCX через **Импорт**.
4. Проверьте рабочий текст: после импорта приложение создаёт JSON в `texts_dm`.
5. Добавьте актёров в проект или выберите их из глобальной базы.
6. Назначьте актёров персонажам в главной таблице.
7. При необходимости переключите область назначения с **Глобально** на **Серия**.
8. Используйте телесуфлёр, отчёты, поиск или экспорт монтажного листа.

## Рабочие тексты

Dubbing Manager не редактирует ASS/SRT напрямую. После импорта создаётся рабочий JSON, который используется для:

- редактирования реплик;
- телесуфлёра;
- глобального поиска;
- статистики проекта и серии;
- HTML/Excel экспорта;
- Reaper RPP экспорта.

Если проект был перенесён, приложение при открытии сначала ищет уже созданные рабочие тексты рядом с проектом, в папке проекта и рядом с исходником. Только если рабочий текст не найден, предлагается создать его заново.

## Актёрские базы

В приложении есть две базы актёров:

- **Проект** — актёры, занятые в текущем проекте.
- **Глобальная база** — общая база актёров между проектами.

Глобальная база хранит имя, цвет и пол актёра. При добавлении актёра в проект он автоматически добавляется в глобальную базу. Если старый проект содержит актёра с тем же именем, что и глобальная база, приложение подтягивает глобальные данные по имени и приводит ID к единому виду.

В настройках есть вкладка **Базы актёров**:

- импорт/экспорт глобальной базы;
- импорт/экспорт распределения актёров проекта;
- перенос актёров между проектом и глобальной базой.

## DOCX

DOCX импорт поддерживает таблицы с настраиваемыми колонками:

- персонаж;
- тайминг начала;
- тайминг конца;
- общий тайминг;
- текст реплики.

DOCX считается уже подготовленным монтажным источником, поэтому при создании рабочего текста строки DOCX не проходят через объединение реплик. Подробнее: [docs/DOCX_IMPORT.md](docs/DOCX_IMPORT.md).

## Экспорт

Доступные форматы:

| Формат | Назначение |
|--------|------------|
| HTML | монтажный лист с настройкой колонок, цветов и редактируемости |
| Excel | табличный монтажный лист и сводка по актёрам |
| Reaper RPP | проект Reaper с треками актёров, регионами и опциональным видео |

В HTML/Excel можно выбрать, показывать тайминг как диапазон или только начало реплики.

## Сборка

Локальная macOS-сборка:

```bash
source .venv/bin/activate
./build.sh
```

Ручная PyInstaller-сборка:

```bash
python scripts/prepare_icons.py
python -m PyInstaller dubbing_manager.spec --clean
```

GitHub Actions:

- `Tests` запускается на push/PR для macOS и Windows.
- `Build Dubbing Manager` запускается вручную или по тегу `v*`, собирает macOS DMG и Windows ZIP.

Для релиза:

```bash
git tag v1.4.0
git push origin v1.4.0
```
