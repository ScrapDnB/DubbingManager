# <img width="64" height="64" alt="Dubbing Manager icon" src="https://github.com/user-attachments/assets/9e226534-92b0-4bd9-b64e-5e082ba229fd" /> Dubbing Manager

**Dubbing Manager** — приложение для подготовки дубляжных проектов: импорт сценариев, рабочие монтажные тексты, распределение ролей по актёрам, телесуфлёр, видео, отчёты и экспорт монтажных листов.

Программа помогает собрать в одном месте то, что обычно разъезжается по субтитрам, таблицам, заметкам и проектам Reaper: кто говорит, кто озвучивает, в какой серии появляется роль, сколько у актёра текста и какие файлы нужны для записи.

## Скачать

Готовые сборки находятся в разделе [Releases](https://github.com/ScrapDnB/DubbingManager/releases).

Актуальная версия: **1.6.0**.

- **macOS**: скачайте `Dubbing_Manager_macOS.dmg`, откройте образ и перенесите приложение в `Applications`.
- **Windows**: скачайте `Dubbing_Manager_Windows.zip`, распакуйте папку и запустите `Dubbing Manager.exe`.

Если система предупреждает, что приложение скачано из интернета, подтвердите запуск стандартным способом для вашей ОС.

## Для чего подходит

- подготовка монтажных листов для дубляжа;
- разбор ASS, SRT и DOCX-сценариев по персонажам;
- назначение актёров на роли во всём проекте или только в конкретной серии;
- проверка, где появляются роли и сколько у актёров реплик;
- телесуфлёр для записи с синхронизацией по времени;
- экспорт HTML/Excel/DOCX/PDF-листов для записи, печати или раздачи команде;
- создание Reaper RPP с регионами и треками актёров;
- перенос кастинга между проектами, сезонами или компьютерами.

## Быстрый старт

1. Запустите Dubbing Manager.
2. Нажмите **Новый проект** или откройте существующий `.json`.
3. В настройках проекта на вкладке **Главные** задайте название проекта и, при необходимости, рабочую папку.
4. Нажмите **Импорт** и выберите ASS, SRT или DOCX.
5. Добавьте актёров в проект.
6. Назначьте актёров персонажам в главной таблице.
7. Откройте **Предпросмотр**, **Телесуфлёр**, **Отчёт серии** или экспортируйте монтажный лист.

После импорта Dubbing Manager создаёт рабочий JSON-текст серии. Дальше программа работает именно с ним: так исходный ASS/SRT/DOCX остаётся источником импорта, а правки, назначения и экспорт не ломают оригинальный файл.

## Основные понятия

### Проект

Проект хранится в `.json` и содержит список серий, актёров, назначений, настроек экспорта, телесуфлёра и путей к файлам.

### Серия

Серия может быть импортирована из:

- `.ass`;
- `.srt`;
- `.docx`.

Для каждой серии можно привязать видео и рабочий текст. Если файлы переехали, окно **Файлы проекта** помогает найти и перепривязать их.

### Рабочий текст

Рабочий текст — это внутренний JSON, который используется для:

- редактирования реплик;
- предпросмотра монтажного листа;
- телесуфлёра;
- поиска по проекту;
- отчётов и статистики;
- HTML/Excel/DOCX/PDF/RPP-экспорта.

Если рабочий текст потерян, его можно пересоздать из исходного ASS/SRT/DOCX в окне **Файлы проекта**.

### Роль

Роль — это персонаж из текста серии. Одну роль можно назначить актёру глобально на весь проект или переопределить назначение для отдельной серии.

## Импорт сценариев

Файлы ASS/SRT можно также быстро перетащить в drop-зону главного окна, чтобы сразу открыть временный предпросмотр монтажного листа без добавления серии в проект. Это удобно для быстрой проверки чужого файла или разового конверта.

### ASS

ASS-импорт берёт персонажа из поля `Name`/`Actor`. Если в поле несколько имён через `;`, Dubbing Manager создаёт отдельную реплику для каждого персонажа.

### SRT

SRT подходит для сценариев, где персонаж указан в тексте или уже подготовлен отдельным способом. После импорта строки попадают в рабочий текст и могут использоваться в монтажных листах.

### DOCX

DOCX-импорт рассчитан на таблицы. В окне импорта можно указать, какие колонки отвечают за:

- персонажа;
- начало тайминга;
- конец тайминга;
- общий тайминг;
- текст реплики.

Перед импортом показывается предпросмотр. Настройки колонок сохраняются, чтобы похожие документы импортировались быстрее.

## Актёры и роли

В приложении есть две базы актёров:

- **Проект** — актёры, которые участвуют в текущем проекте. Здесь хранятся цвета, пол и назначения.
- **Глобальная база** — общая адресная книга актёров между проектами. В ней хранятся имя и пол. Цвета в глобальную базу не переносятся, потому что цвет — настройка конкретного проекта.

### Назначение актёров

Назначить актёра можно в главной таблице. Обычно используется глобальное назначение: персонаж всегда озвучивается одним актёром. Если в конкретной серии роль временно озвучивает другой актёр, переключите назначение на уровень серии и задайте локальное переопределение.

### Окно «Роли проекта»

В настройках проекта на вкладке **Главные** есть кнопка **Роли проекта**. В этом окне можно:

- увидеть все роли проекта;
- посмотреть, в каких сериях роль появляется;
- найти роль по имени персонажа или актёра;
- переназначить роль другому актёру;
- сбросить привязку роли к актёру, не удаляя персонажа из текстов серий.

### Перенос кастинга

Во вкладке **Базы актёров** можно экспортировать и импортировать распределение актёров проекта. Это удобно, если нужно перенести кастинг в новый сезон или похожий проект.

## Главная таблица

Главная таблица показывает персонажей текущей серии и помогает быстро контролировать кастинг.

Полезные элементы:

- фильтр по актёру;
- фильтр неназначенных персонажей;
- поиск;
- цветовая подсветка актёров;
- переключение между назначением на весь проект и назначением только для серии;
- статистика по выбранному персонажу.

## Настройки проекта

Окно **Настройки проекта** содержит несколько вкладок:

- **Главные** — название проекта, автор, студия, рабочая папка, файлы проекта и роли проекта.
- **Экспорт** — вид монтажного листа, колонки, размеры шрифтов, подсветка, округление времени и режим тайминга.
- **Объединение** — правила объединения коротких реплик.
- **Телесуфлёр** — размеры текста, синхронизация, порты OSC, offset и цвета.
- **DOCX** — настройки импорта документов.
- **Базы актёров** — импорт и экспорт глобальной базы и распределения актёров проекта.

## Предпросмотр монтажного листа

Предпросмотр показывает HTML-лист до экспорта. В нём можно менять:

- формат: **Таблица** или **Сценарий**;
- видимые колонки;
- режим тайминга: начало и конец или только начало;
- округление времени;
- размеры шрифтов;
- актёров для цветовой подсветки.

Изменения сразу применяются к предпросмотру.

## Экспорт

### HTML

HTML подходит для просмотра в браузере, записи с экрана или раздачи команде. Доступны две разметки:

- **Таблица** — компактный монтажный лист с колонками времени, персонажа, актёра и текста.
- **Сценарий** — более свободная разметка реплик блоками.

Обе разметки учитывают настройки тайминга: округление, диапазон или только начало реплики.

### Excel

Excel-экспорт создаёт:

- листы по сериям;
- сводку по актёрам;
- колонки с таймингом, персонажем, актёром и репликой;
- цветовую подсветку актёров, если она включена.

### DOCX

DOCX-экспорт создаёт монтажный лист для Word или Pages. Он использует те же настройки колонок, тайминга, размеров и подсветки, что и предпросмотр.

### PDF

PDF-экспорт создаёт готовый к отправке или печати монтажный лист на основе той же HTML-разметки, что используется в предпросмотре.

### Reaper RPP

RPP-экспорт создаёт проект Reaper с регионами по репликам и треками актёров. Перед экспортом показывается предпросмотр: сколько будет регионов, треков и будет ли добавлено видео.

## Телесуфлёр

Телесуфлёр показывает реплики текущей серии крупным текстом и помогает записывать озвучку.

Возможности:

- автопрокрутка по времени;
- ручная прокрутка без потери текущей серии;
- фильтр актёров;
- обновление каста без закрытия окна;
- зеркальный режим;
- настройка размеров и цветов;
- OSC-синхронизация с Reaper.

Если включена синхронизация с Reaper, телесуфлёр может следовать за таймкодом проекта. Offset применяется к телесуфлёру, но не меняет экспорт регионов RPP.

## Видео

К серии можно привязать видео. Рабочая папка проекта помогает Dubbing Manager находить видео после переноса проекта на другой диск или компьютер.

Поддерживаемые расширения:

- `.mp4`;
- `.mkv`;
- `.avi`;
- `.mov`;
- `.m4v`;
- `.wmv`.

## Проверка проекта

Кнопка проверки проекта помогает найти типовые проблемы:

- нет серий;
- отсутствуют исходные файлы;
- не найдено видео;
- нет рабочих текстов;
- есть персонажи без назначенного актёра;
- есть битые пути после переноса проекта.

## Поиск и отчёты

Dubbing Manager умеет:

- искать текст по всем сериям;
- показывать отчёт по серии;
- показывать сводку проекта;
- считать реплики и слова по ролям и актёрам;
- показывать, в каких сериях встречаются роли.

## Обновления

Проверить обновления можно в окне **О программе**. Если доступна новая версия, приложение предложит открыть релиз или установить обновление автоматически, если такая установка поддерживается для вашей платформы.

На macOS принудительную проверку можно запустить с зажатой **Option**, на Windows — с зажатой **Alt**.

## Форматы файлов

| Что | Форматы |
| --- | --- |
| Источники сценария | `.ass`, `.srt`, `.docx` |
| Проект | `.json` |
| Рабочие тексты | `.json` в папке `texts_dm` |
| Видео | `.mp4`, `.mkv`, `.avi`, `.mov`, `.m4v`, `.wmv` |
| Экспорт | `.html`, `.xlsx`, `.docx`, `.pdf`, `.rpp` |

## Советы по работе

- Сначала задайте рабочую папку проекта, если файлы лежат в одной структуре папок.
- После импорта проверьте окно **Файлы проекта**: там видно, какие исходники, рабочие тексты и видео привязаны к сериям.
- Для больших проектов удобнее вести актёров в глобальной базе, а цвета назначать отдельно в каждом проекте.
- Если персонаж в одной серии озвучивается другим актёром, используйте серийное назначение, а не переименование роли.
- Перед экспортом откройте предпросмотр и проверьте колонки, тайминг, округление и подсветку.
- Если проект перенесли, не импортируйте всё заново сразу: сначала попробуйте восстановить пути через рабочую папку и окно файлов проекта.

## Запуск из исходников

Этот способ нужен, если вы хотите запустить приложение без готовой сборки.

```bash
git clone https://github.com/ScrapDnB/DubbingManager.git
cd DubbingManager
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python main.py
```
