# Changelog

## [1.4.0] - 2026-05-02

### Главное

- Добавлена история недавних проектов и быстрый доступ к ним из верхней панели.
- Добавлен быстрый фильтр главной таблицы по актёру и отдельный фильтр пустых назначений рядом с поиском.
- Добавлена глобальная база актёров: её можно вести отдельно от проекта, импортировать, экспортировать и переносить актёров между глобальной базой и проектом.
- Добавлен импорт и экспорт распределения актёров, чтобы переносить кастинг между проектами или сезонами.
- Добавлен предпросмотр Reaper RPP прямо в окне выбора параметров экспорта.
- Добавлены автобэкапы рабочих текстов и глобальных настроек перед перезаписью.

### Добавлено

- Пол актёра (`М`/`Ж`) в глобальной базе и в проектной базе актёров.
- Сортировка и пометки в глобальной базе актёров: видно, кто уже добавлен в текущий проект.
- Пакетное добавление актёров из проекта в глобальную базу с пропуском уже существующих актёров.
- Синхронизация актёров проекта с глобальной базой по имени: если актёр уже есть в глобальной базе, подтягиваются его цвет, пол и единый ID.
- Перенос глобальной базы актёров и распределения проекта в отдельную вкладку настроек «Базы актёров».
- Возможность показывать в монтажном листе только тайминг начала реплики без тайминга конца.
- Настройки колонок и тайминга прямо в окне предпросмотра монтажного листа.
- Использование нового набора иконок приложения при локальной сборке и в GitHub Actions.

### Изменено

- Верхняя панель стала компактнее: часть кнопок заменена иконками, кнопка видео стала короче, «Отчёт серии» перенесён в инструменты.
- Добавлена отдельная кнопка создания нового проекта; она предупреждает о несохранённых изменениях.
- Главная таблица переведена на более быстрый табличный вид, чтобы прокрутка и изменение размера окна были плавнее.
- Комбобоксы назначения актёров в таблице стали выглядеть нативнее и больше не перекрываются текстом ячейки.
- В окне файлов проекта DOCX теперь отображается как исходный файл серии, даже если серия уже работает через JSON.
- Пересоздание рабочего текста теперь работает не только из ASS/SRT, но и из DOCX.
- Рабочий текст из DOCX создаётся без объединения реплик: строки документа сохраняются как подготовленные реплики.
- При открытии проекта приложение сначала ищет уже существующие рабочие тексты рядом с проектом и в папке проекта, и только потом предлагает их создать.
- Табличный HTML-экспорт и Excel-экспорт теперь учитывают выбранные колонки и настройки тайминга.

### Исправлено

- Исправлены случаи, когда при открытии проекта программа предлагала заново создать уже существующие рабочие тексты.
- Исправлена потеря отображения исходного DOCX после перехода серии на рабочий JSON.
- Исправлено отображение пола актёров в старых проектах, где ID актёра отличался от ID в глобальной базе.
- Исправлена обрезка иконок на кнопках верхней панели.
- Исправлены проблемы с лишними локальными служебными файлами в публичном репозитории.

### Сборка и релизы

- Версия приложения обновлена до `1.4.0`.
- Обновлены macOS и Windows иконки приложения.
- Локальные служебные файлы Codex и coverage-отчёты удалены из отслеживания Git и добавлены в `.gitignore`.

### Тесты

- Проверка перед коммитом: `587 passed, 11 skipped`.

## [1.3.1] - 2026-05-01

- Добавлена проверка проекта: приложение помогает найти отсутствующие файлы, проблемы с рабочими текстами, видео и неназначенными персонажами.
- Главная таблица стала легче и отзывчивее при прокрутке, изменении размера окна и редактировании назначений.

### Добавлено

- Новая иконка приложения для macOS и Windows.

- Описания к настройкам, особенно к параметрам экспорта.

### Исправлено

- Исправлены цвета верхней панели телесуфлёра в светлой системной теме.
- При редактировании актёра в главной таблице текст ячейки больше не накладывается на выпадающий список.
- Комбобоксы в главной таблице выглядят более нативно.
- Возвращена нормальная адаптация таблицы к изменению размера окна.

## [1.3.0] - 2026-04-30

### Главное

- Рабочий текст серии теперь хранится в отдельном JSON и становится основным источником правок после импорта.
- ASS/SRT/DOCX используются как форматы импорта, а не как редактируемый источник истины.
- Телесуфлёр, предпросмотр, экспорт и статистика читают данные из рабочего JSON-текста.
- Добавлена миграция старых проектов и совместимость с проектами без новых полей.

### Добавлено

- Генерация рабочего JSON-текста для серий при импорте ASS/SRT/DOCX.
- `ScriptTextService` для чтения, записи и миграции рабочих текстов.
- `AssignmentService` для централизованной работы с назначениями персонажей и актёров.
- Возможность создать рабочий JSON из субтитров через менеджер файлов проекта, если рабочий текст был потерян.
- Расширенный просмотр ролей актёра.
- Статистика по персонажу.
- Новые настройки импорта DOCX и сохранение этих настроек.
- Иконка приложения для macOS (`.icns`) и Windows (`.ico`).
- GitHub Actions сборка Windows и macOS артефактов.
- Версия приложения `1.3.0`; версия формата проекта `1.3`.

### Изменено

- Редактирование текста перенесено с ASS/SRT-файлов на рабочие JSON-файлы.
- Импорт DOCX адаптирован под новую архитектуру рабочих текстов.
- Экспорт монтажных листов и HTML использует рабочие тексты серий.
- Экспорт RPP в Reaper теперь идёт через единую реализацию в `ExportService`.
- macOS-сборка переведена на `onedir` внутри `.app`, что ускоряет запуск по сравнению с прежним вариантом.
- Импорты QtWebEngine и QtMultimedia сделаны ленивыми, чтобы приложение быстрее открывалось.
- Windows-сборка в CI собирается как `onedir` ZIP, а не как один большой `onefile` executable.
- About-окно теперь берёт версию из общей константы `APP_VERSION`.

### Исправлено

- Ручной скролл телесуфлёра больше не застревает на выбранной реплике при отключённой синхронизации с Reaper.
- При включённой синхронизации с Reaper пользователь может временно свободно скроллить телесуфлёр вручную.
- Экспорт регионов Reaper больше не создаёт регионы нулевой длины: регион начинается по началу фразы и заканчивается по её концу.
- Offset телесуфлёра больше не применяется к созданию регионов Reaper.
- HTML-экспорт экранирует пользовательский текст и не ломается на спецсимволах или вставленном HTML.
- Excel-экспорт больше не падает на нечисловых номерах серий вроде `1A`, `S01E02` или `pilot`.
- Batch Excel export больше не пересоздаёт общий workbook для каждой серии.
- Конфиги новых проектов и глобальных настроек теперь копируются глубоко, без разделяемых вложенных словарей.
- Повторная локальная macOS-сборка стабильнее очищает старые build/dist артефакты.
- В `.app` корректно попадают `CFBundleVersion`, `CFBundleShortVersionString` и иконка приложения.

### Сборка и релизы

- Обновлён `dubbing_manager.spec`:
  - macOS собирает `.app` с `DubbingManager.icns`;
  - Windows собирает папку приложения с `Dubbing Manager.exe` и `DubbingManager.ico`;
  - версия сборки берётся из `APP_VERSION`.
- Обновлён `.github/workflows/build.yml`:
  - Windows artifact: `Dubbing_Manager_Windows.zip`;
  - macOS artifact: `Dubbing_Manager_macOS.dmg`;
  - на тегах `v*` артефакты прикрепляются к GitHub Release.
- Обновлена инструкция `.github/BUILD.md`.

### Тесты

- Добавлены тесты для рабочих текстов, миграции старых проектов, DOCX-импорта, назначений актёров, телесуфлёра и экспорта.
- Локальная проверка перед changelog: `548 passed, 11 skipped`.

#### Атомарное сохранение:
```python
temp_path = path + ".tmp"
with open(temp_path, 'w') as f:
    json.dump(data, f)
    f.flush()
    os.fsync(f.fileno())
os.replace(temp_path, path)  # Атомарная замена
```

def _validate_project_structure(data: Dict[str, Any]) -> None:
