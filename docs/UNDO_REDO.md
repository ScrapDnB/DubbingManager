# Система отмены/повтора действий (Undo/Redo)

## Обзор

В приложение Dubbing Manager добавлена полноценная система отмены и повтора действий. Теперь вы можете отменять ошибочные действия и возвращать их обратно.

## Возможности

### Поддерживаемые операции отмены:
- **Добавление актёра** - добавление нового актёра в базу
- **Удаление актёра** - удаление актёра из базы (включая все назначения)
- **Переименование актёра** - изменение имени актёра
- **Изменение цвета актёра** - изменение цвета актёра
- **Назначение актёра на персонажа** - привязка актёра к персонажу
- **Переименование персонажа** - изменение имени персонажа в таблице
- **Добавление эпизода** - импорт ASS файла
- **Переименование эпизода** - изменение номера/названия серии
- **Удаление эпизода** - удаление серии из проекта
- **Переименование проекта** - изменение названия проекта

## Использование

### Удаление актёра:
1. Выберите актёра в таблице "БАЗА АКТЕРОВ"
2. Нажмите кнопку **"- Актер"**
3. Подтвердите удаление в диалоге

> ⚠️ **Внимание:** При удалении актёра все его назначения на персонажей будут удалены.

### Кнопки Undo/Redo:
Кнопки расположены в **верхней панели** окна, рядом с кнопками управления проектом:
- **↶ Отмена** — отменить последнее действие
- **↷ Повтор** — повторить отменённое действие

### Горячие клавиши:
- **Ctrl+Z** - отменить последнее действие
- **Ctrl+Shift+Z** - повторить отменённое действие

### Кнопки в интерфейсе:
В правой панели инструментов добавлены кнопки:
- **↶ Отмена** - отменить последнее действие
- **↷ Повтор** - повторить отменённое действие

Кнопки активны только когда доступны соответствующие операции.

## Архитектура

### Базовые классы (`core/commands.py`):

#### `Command` (абстрактный класс)
Базовый класс для всех команд. Каждая команда должна реализовать:
- `execute()` - выполнение действия
- `undo()` - отмена действия
- `get_description()` - текстовое описание для UI

#### `UndoStack`
Стек для хранения и управления командами:
- `push(command)` - выполнить и сохранить команду
- `undo()` - отменить последнее действие
- `redo()` - повторить отменённое действие
- `clear()` - очистить историю
- `can_undo()` - проверка возможности отмены
- `can_redo()` - проверка возможности повтора

### Реализованные команды:

| Команда | Описание |
|---------|----------|
| `AddActorCommand` | Добавление актёра |
| `DeleteActorCommand` | Удаление актёра |
| `RenameActorCommand` | Переименование актёра |
| `UpdateActorColorCommand` | Изменение цвета актёра |
| `AssignActorToCharacterCommand` | Назначение актёра на персонажа |
| `RenameCharacterCommand` | Переименование персонажа |
| `AddEpisodeCommand` | Добавление эпизода |
| `RenameEpisodeCommand` | Переименование эпизода |
| `DeleteEpisodeCommand` | Удаление эпизода |
| `UpdateProjectNameCommand` | Переименование проекта |

## Особенности реализации

### Автоматическое управление состоянием
- При загрузке нового проекта стек отмены очищается
- Каждое новое действие автоматически очищает стек Redo
- Максимальный размер стека - 100 команд (предотвращает переполнение памяти)

### Интеграция с существующим кодом
- Все изменения в данных теперь проходят через команды
- UI автоматически обновляется после выполнения/отмены команд
- Флаг `is_dirty` устанавливается автоматически

### Тестирование
Все команды покрыты модульными тестами в файле `tests/test_undo_stack.py`.

Запуск тестов:
```bash
python -m pytest tests/test_undo_stack.py -v
```

## Пример использования

```python
# В MainWindow
from core.commands import AddActorCommand, UndoStack

# Создание стека
self.undo_stack = UndoStack()

# Выполнение действия через команду
actor_id = str(datetime.now().timestamp())
command = AddActorCommand(
    self.data["actors"],
    actor_id,
    "Имя Актёра",
    "#FFFFFF"
)
self.undo_stack.push(command)

# Отмена последнего действия
self.undo_stack.undo()

# Повтор отменённого действия
self.undo_stack.redo()
```

## Расширение функциональности

Для добавления новой команды:

1. Создайте новый класс команды, наследуясь от `Command`:
```python
class MyNewCommand(Command):
    def __init__(self, data, ...):
        self.data = data
        # Сохраните параметры для отмены
        
    def execute(self):
        # Код выполнения действия
        pass
        
    def undo(self):
        # Код отмены действия
        pass
        
    def get_description(self):
        return "Описание действия"
```

2. Добавьте импорт команды в `core/__init__.py`

3. Используйте команду в `MainWindow`:
```python
command = MyNewCommand(self.data, ...)
self.undo_stack.push(command)
self.set_dirty()
```

## Известные ограничения

- Некоторые операции (например, экспорт файлов) не поддерживают отмену
- Отмена переименования эпизода может потребовать дополнительного обновления UI
- При закрытии приложения история отмены не сохраняется
