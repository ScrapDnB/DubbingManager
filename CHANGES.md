# Изменения: Исправление функции выбора цветов в настройках экспорта

## Проблема 1: Галочка "Цвета актёров" не работала
- Галочка в диалоге `ExportSettingsDialog` не влияла на экспорт
- Монтажный лист всегда экспортировался полностью покрашенным, независимо от состояния галочки
- Не было механизма выбора конкретных актёров для подсветки при экспорте (в отличие от предпросмотра)

## Проблема 2: Диалог о несохранённых изменениях в окне предпросмотра
- При закрытии `HtmlLivePreview` появлялся диалог "Несохраненные изменения", даже если пользователь ничего не менял
- Причина: изменения размеров шрифтов в предпросмотре устанавливали флаг `main_app.is_dirty`, который проверялся при закрытии

## Решение 1: Исправление выбора цветов

### Модификация `ExportSettingsDialog` (класс)
- **Добавлены**: переменные `self.parent_app` и `self.highlight_ids_export` для хранения родительского приложения и выбранных актёров
- **Переорганизована**: секция галочек - теперь `use_color` и выбор актёров находятся в отдельной группе "Отображение цветов"
- **Добавлена**: кнопка "Выбрать актёров для подсветки..." которая открывает `ActorFilterDialog` (аналогично окну предпросмотра)
- **Создан метод**: `open_actor_filter_for_export()` для управления выбором актёров
- **Обновлен метод**: `get_settings()` теперь сохраняет `highlight_ids_export` в конфиг экспорта

### Модификация функции `generate_html_body()`
- **Добавлена**: проверка параметра `use_color` из конфига (по умолчанию `True`)
- **Реализована**: логика применения цветов:
  - Если `use_color = False` → цвета не применяются (всё белое)
  - Если `use_color = True` && выбраны конкретные актёры → только выбранные актёры подсвечиваются
  - Если `use_color = True` && строка "все" → все актёры подсвечиваются

### Модификация функции `_create_excel_book()`
- **Добавлен параметр**: `cfg` (конфигурация экспорта)
- **Реализована**: та же логика применения цветов для Excel файлов
- **Обновлены** все вызовы этой функции

### Обновлены функции экспорта
- `export_to_html()` - передаёт `highlight_ids_export` в `generate_html_body()`
- `export_to_excel()` - передаёт конфиг в `_create_excel_book()`
- `_execute_batch_export()` - использует оба параметра при массовом экспорте

## Решение 2: Исправление диалога о несохранённых изменениях

### Модификация `HtmlLivePreview`
- **Добавлена переменная**: `self._has_text_changes` для отслеживания только реальных изменений текста
- **Обновлен метод**: `on_setting_change()` - больше не устанавливает флаг грязности главного приложения (так как это только изменения отображения, а не текста)
- **Обновлен метод**: `closeEvent()` - проверяет только `self._has_text_changes`, а не глобальный флаг `main_app.is_dirty`
- **Обновлены методы**: `save_to_original_ass()` - очищает флаг `_has_text_changes` при сохранении

### Модификация `WebBridge`
- **Обновлен метод**: `update_text()` - теперь устанавливает флаг `preview_window._has_text_changes = True` когда пользователь реально редактирует текст

## Результат
✅ Настройки экспорта полностью независимы от предпросмотра  
✅ Галочка "Использовать цвета актёров" действительно работает  
✅ Можно выбрать конкретных актёров для подсветки при экспорте  
✅ Выбор в окне предпросмотра не влияет на экспорт  
✅ При экспорте используются только настройки из `ExportSettingsDialog`  
✅ При закрытии окна предпросмотра диалог о сохранении появляется **только если** пользователь действительно редактировал текст репlik  
✅ Работает для HTML и Excel форматов
